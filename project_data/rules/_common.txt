РЕПОЗИТОРИИ
===========

• Обязательные ветки:
"master" — только для версий;
"dev" — только для готовых этапов.

• Каждый этап делается в отдельной ветке, отделённой от ветки "dev".

• Ветка "dev" должна содержать только действующий отлаженный код.

• Все промежуточные снимки должны быть атомарными.

• Форматирование кода желательно выполняется в последнем снимке перед слиянием в ветку "dev".

• Названия снимков в ветке "dev" используются для автоматического составления "changelog.*" и должны соответствовать его требованиям.

• Все необходимые для версии изменения (номер версии и подобные) делаются в снимке слияния с веткой "master" с параметрами "--no-ff --no-commit".


БЕЗОПАСНОСТЬ
============

• Все конфиденциальные данные хранятся на личных носителях команды Афотериса и распределены между её членами по устной договорённости.

• Пароли должны быть из 16-ти символов и обязательно содержать цифры, строчные и заглавные буквы, знак «+» и/или «-».

• Префиксы для таблиц БД должны быть 4-х значными и обязательно иметь цифры и буквы.


ДОКУМЕНТАЦИЯ
============


ОБЩИЕ СОГЛАШЕНИЯ

• Вся документация ведётся на "html".
• Простые соглашения допускается вести в простых текстовых файлах с расширением "txt".
• Стилевое оформление должно быть стандартизировано (единый файл "docstyle.css") и применяется только к тегам, произвольные классы запрещены.
• Граница стандартизации оформления тег "body".
• Специальные символы представлять html-сущностями.


СПИСОК СООТВЕТСТВИЯ

• body — содержимое.
• header — заглавная часть.
• h1—h6 — заголовки.
• p — абзацы.
• ol, ul, li — списки.
• link — ссылки.
• strong — выделение жирным.
• br — перенос строки.
