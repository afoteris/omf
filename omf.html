Об ОМФ

ОМФ — это отобранное подмножество стандартных тегов метаданных фотографий (далее — теги) для снабжения основных пользовательских административных, описательных и авторских потребностей.

ОМФ представлены в виде двух видов списков:

  • Постандартные списки тегов из соответствующих стандартов метаданных;
  • Объединённый список тегов, который создан специально для удобства как единый источник этих тегов.

Теги для ОМФ взяты из следующих стандартов:

  • CIPA DC-010-2020 (Exif 2.32 metadata for XMP)
  • IPTC Photo Metadata Standard 2021.1 (IPTC Core schema 1.3)
  • IPTC Photo Metadata Standard 2021.1 (IPTC Extension schema 1.6)
  • PLUS Version 1.2.3
  • XMP (XMP SPECIFICATION PART 1. DATA MODEL, SERIALIZATION, AND CORE PROPERTIES. April 2012)
  • XMP (XMP SPECIFICATION PART 2. ADDITIONAL PROPERTIES (XMP Toolkit SDK v2022.06))

Постандартные списки тегов:

  • составлены индивидуально по каждому из используемых стандартов;
  • сопутствующая информация из стандартов для однозначности представлена без перевода.

Объединённый список тегов:

  • составлен из Постандартных списков тегов с сокращением возможных повторений из-за пересечения этих списков между собой;
  • дополнительно каждому стандартному тегу сопоставлен соответствующий ExifTool-тег;
  • знаком => указаны ExifTool-теги, которые в основе своего названия отличаются от названий стандартных тегов.

При использовании ExifTool для управления структурами метаданных, достаточно управлять только дочерними тегами этих структур, сами структуры при необходимости ExifTool достраивает автоматически.

Property PLUS Version

XMP Value Type Text
Description The version number of the PLUS standards in place at the time of the transaction.
Comments In the event of a misunderstanding between the parties, PLUS Version provides a reference to the version of the PLUS standards that was in place at the time of the transaction. PLUS standards are updated periodically.
Format x.x.x
Example 1.2.0
Notes 1. Version is made up of three components: Major Version, Minor Version and Revision. Each of these is combined with periods to form a single version number (e.g. 1.2.3 means Major Version 1 Minor Version 2 Revision 3). A Major Version will be published every two years when the standards are reviewed. Minor Versions may be published in between where necessary and will require re-certification. Finally, Revisions are minor changes to the standards that do not require re-certification.

XMP SPECIFICATION PART 1. DATA MODEL, SERIALIZATION, AND CORE PROPERTIES. April 2012

Name dc:creator
Type Ordered array of ProperName
Property content DCMI definition: An entity primarily responsible for making the resource.
DCMI comment: Examples of a creator include a person, an organization, or a service. Typically, the name of a creator should be used to indicate the entity.
XMP addition: XMP usage is a list of creators. Entities should be listed in order of decreasing precedence, if such order is significant.
Name dc:description
Type Language Alternative
Property content DCMI definition: An account of the resource.
XMP addition: XMP usage is a list of textual descriptions of the content of the resource, given in various languages.
Name dc:rights
Type Language Alternative
Property content DCMI definition: Information about rights held in and over the resource.
DCMI comment: Typically, rights information includes a statement about various property rights associated with the resource, including intellectual property rights.
XMP addition: XMP usage is a list of informal rights statements, given in various languages.
Name dc:subject
Type Unordered array of Text
Property content DCMI definition: The topic of the resource.
DCMI comment: Typically, the subject will be represented using keywords, key phrases, or classification codes. Recommended best practice is to use a controlled vocabulary. To describe the spatial or temporal topic of the resource, use the dc:coverage element.
XMP addition: XMP usage is a list of descriptive phrases or keywords that specify the content of the resource.
Name dc:title
Type Language Alternative
Property content DCMI definition: A name given to the resource.
DCMI comment: Typically, a title will be a name by which the resource is formally known.
XMP addition: XMP usage is a title or name, given in various languages
Name xmp:CreateDate
Type Date
Property content The date and time the resource was created. For a digital file, this need not match a file-system creation time. For a freshly created resource, it should be close to that time, modulo the time taken to write the file. Later file transfer, copying, and so on, can make the file-system time arbitrarily different.
Name xmp:CreatorTool
Type AgentName
Property content The name of the first known tool used to create the resource.
Name xmp:Label
Type Text
Property content A word or short phrase that identifies a resource as a member of a user-defined collection.
NOTE One anticipated usage is to organize resources in a file browser.
Name xmpRights:Marked
Type Boolean
Property content When true, indicates that this is a rights-managed resource. When false, indicates that this is a public-domain resource. Omit if the state is unknown
Name xmpRights:Owner
Type Unordered array of ProperName
Property content A list of legal owners of the resource
Name xmpRights:UsageTerms
Type Language Alternative
Property content A collection of text instructions on how a resource can be legally used, given in a variety of languages.
Name xmpMM:DocumentID
Type GUID
Property content The common identifier for all versions and renditions of a resource. See Annex A, “(informative) Document and instance IDs” and definitions of rendition (3.7) and version (3.9).
Name xmpMM:OriginalDocumentID
Type GUID
Property content The common identifier for the original resource from which the current resource is derived. For example, if you save a resource to a different format, then save that one to another format, each save operation should generate a new xmpMM:DocumentID that uniquely identifies the resource in that format, but should retain the ID of the source file here.
See Annex A, “(informative) Document and instance IDs”.
