Об ОМФ

ОМФ — это отобранное подмножество стандартных метаданных для снабжения основных потребностей жизненного цикла фотографий, а именно:

  • уникальное именование (в соответствии с ПИФ);
  • хранение информации о дате и времени съёмки;
  • хранение информации об авторстве;
  • хранение информации об авторских правах;
  • хранение информации о владельце устройства, с помощью которого была создана фотография;
  • хранение описания визуальной составляющей фотографии.
  • хранение необходимой дополнительной информации.

ОМФ представлены в виде объединённого списка тегов (Объединённый список тегов) из нескольких соответствующих стандартов метаданных (Постандартные списки тегов), который создан специально для удобства как единый источник этих тегов.

Теги для ОМФ взяты из следующих стандартов:

  • CIPA DC-010-2020 (Exif 2.32 metadata for XMP)
  • IPTC Photo Metadata Standard 2021.1 (IPTC Core schema 1.3)
  • IPTC Photo Metadata Standard 2021.1 (IPTC Extension schema 1.6)
  • PLUS Version 1.2.3
  • XMP (XMP SPECIFICATION PART 1. DATA MODEL, SERIALIZATION, AND CORE PROPERTIES. April 2012)
  • XMP (XMP SPECIFICATION PART 2. ADDITIONAL PROPERTIES. August 2016)

Примечания взяты из стандартов и представлены без перевода.

Дополнительно каждому стандартному тегу метаданных сопоставлен ExifTool-тег.

Знаком => указаны ExifTool-теги, которые в основе своего названия отличаются от названий стандартных тегов.

При использовании ExifTool для добавления структур метаданных, достаточно добавлять только дочерние поля, родительские поля ExifTool создаёт и добавляет автоматически.

Постандартные списки тегов

Эти списки составлены индивидуально по каждому из представленных стандартов, не учитывая возможные повторения из-за пересечения списков между собой.

CIPA DC-010-2020

Exif 2.32 metadata for XMP

Родной ImageDescription
XMP-аналог dc:description
Примечание The title of the image as an ASCII string.
Родной Model
XMP-аналог tiff:Model
Примечание Model name or number of the equipment as an ASCII string.
Родной Artist
XMP-аналог dc:creator
Примечание Camera owner, photographer or image creator as an ASCII string.
Родной Copyright
XMP-аналог dc:rights
Примечание Copyright information as an ASCII string.
Родной UserComment
XMP-аналог exif:UserComment
Примечание Comments from user.
Родной DateTimeOriginal
XMP-аналог exif:DateTimeOriginal
Примечание Date and time when the original data was generated (no time zone in Exif), stored in ISO 8601 format, not in the original Exif format. This property includes the value for the Exif SubSecTimeOriginal (37521, 0x9291) attribute.
Родной OffsetTimeOriginal
XMP-аналог -
Примечание Offset from UTC of the time of DateTimeOriginal.
Родной FileSource
XMP-аналог exif:FileSource
Примечание Indicates image source:
0 = others
1 = scanner of transparent type
2 = scanner of reflex type
3 = DSC
Родной CameraOwnerName
XMP-аналог exifEX:CameraOwnerName
Примечание This tag records the owner of a camera used in photography as an ASCII string.
Родной BodySerialNumber
XMP-аналог exifEX:BodySerialNumber
Примечание The serial number of the camera or camera body used to take the photograph.

Родной Title

XMP

XMP SPECIFICATION PART 1. DATA MODEL, SERIALIZATION, AND CORE PROPERTIES. April 2012

Родной dc:creator
Примечание DCMI definition: An entity primarily responsible for making the resource.
DCMI comment: Examples of a creator include a person, an organization, or a service. Typically, the name of a creator should be used to indicate the entity.
XMP addition: XMP usage is a list of creators. Entities should be listed in order of decreasing precedence, if such order is significant.
Родной dc:description
Примечание DCMI definition: An account of the resource.
XMP addition: XMP usage is a list of textual descriptions of the content of the resource, given in various languages.
Родной dc:identifier
Примечание DCMI definition: An unambiguous reference to the resource within a given context.
DCMI comment: Recommended best practice is to identify the resource by means of a string conforming to a formal identification system
Родной dc:rights
Примечание DCMI definition: Information about rights held in and over the resource.
DCMI comment: Typically, rights information includes a statement about various property rights associated with the resource, including intellectual property rights.
XMP addition: XMP usage is a list of informal rights statements, given in various languages.
Родной dc:subject
Примечание DCMI definition: The topic of the resource.
DCMI comment: Typically, the subject will be represented using keywords, key phrases, or classification codes. Recommended best practice is to use a controlled vocabulary. To describe the spatial or temporal topic of the resource, use the dc:coverage element.
XMP addition: XMP usage is a list of descriptive phrases or keywords that specify the content of the resource.
Родной dc:title
Примечание DCMI definition: A name given to the resource.
DCMI comment: Typically, a title will be a name by which the resource is formally known.
XMP addition: XMP usage is a title or name, given in various languages.
Родной xmp:CreateDate
Примечание The date and time the resource was created. For a digital file, this need not match a file-system creation time. For a freshly created resource, it should be close to that time, modulo the time taken to write the file. Later file transfer, copying, and so on, can make the file-system time arbitrarily different.
Родной xmp:CreatorTool
Примечание The name of the first known tool used to create the resource
Родной xmp:Label
Примечание A word or short phrase that identifies a resource as a member of a user-defined collection.
NOTE One anticipated usage is to organize resources in a file browser.
Родной xmpRights:Marked
Примечание When true, indicates that this is a rights-managed resource. When false, indicates that this is a public-domain resource. Omit if the state is unknown.
Родной xmpRights:Owner
Примечание A list of legal owners of the resource.
Родной xmpRights:UsageTerms
Примечание A collection of text instructions on how a resource can be legally used, given in a variety of languages.
Родной xmpMM:DocumentID
Примечание Created once for new resources. Different renditions are expected to have different values for xmpMM:DocumentID.
Родной xmpMM:OriginalDocumentID
Примечание Links a resource to its original source. For example, when you save a PSD document as a JPEG, then convert the JPEG to GIF format, the immediate source of the GIF is the JPEG, and the original source is the PSD. The value of xmpMM:OriginalDocumentID in both the JPEG and GIF files is the value of xmpMM:DocumentID from the original PSD file.

XMP SPECIFICATION PART 2. ADDITIONAL PROPERTIES. August 2016

Родной xmp:Nickname
Примечание A short informal name for the resource.
Родной xmpMM:VersionID
Примечание The document version identifier for this resource.
Each version of a document gets a new identifier, usually simply by incrementing integers 1, 2, 3 . . . and so on.
Media management systems can have other conventions or support branching which requires a more complex scheme.
Родной xmpDM:cameraModel
Примечание The make and model of the camera used for a shoot.
Родной xmpDM:comment
Примечание A user’s comments.
Родной crs:RawFileName
Примечание File name for raw file (not a complete path).
